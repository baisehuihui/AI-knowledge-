# 爆文拆解案例04（第1天补）：AI产品黄叔·以招聘为内容的写法

**说明**：这是第1天欠的5篇精拆中的第1篇补拆。之前已拆过黄叔的文章（案例04），本篇重点补充之前遗漏的另一个手法维度。

**来源**：AI产品黄叔公众号（通往AGI之路团队）
**学习主题**：**观点表达**——如何用"对话体"传递态度

---

## 核心手法：**示弱开场 + 对话体推进 + 收尾反转**

### 开头手法
> "说实话，看完这个需求我第一反应是：这活儿真的能用AI干吗？"

**手法**：**示弱开场**。作者不装，直接说"我不确定"。这和"我给大家分享一个干货"的专家姿态完全相反。示弱让读者放下防御，愿意听你继续说。

**观点表达上的意义**：观点不是"我说的都对"才有力，有时候"我也有疑惑"反而更真实、更有说服力。

### 观点表达：没有"我认为"，只有"我发现"

全文几乎没有"我认为""我想强调"这类句子，观点全部藏在下面几种句式里：

1. **场景式观点**：用具体场景带出态度
   - 不是"AI很重要" → 是"凌晨两点还在手动复制粘贴AI回复"
   - 不是"要系统化思考" → 是"他的竞争对手已经搭建了一套全自动工作流"

2. **数据式观点**：用数据来暗示立场
   - 不是"AI会影响很多人" → 是"某电商运营3分钟出稿vs手动复制粘贴"
   
3. **他人之口式观点**：借用角色的话来表达
   - 不直接说"技术有局限" → 让工程师说"现在能做到这个程度已经不错了"

### 节奏感：短句+空拍

"先试了第一版。
不行。
再试第二版。
还是不行。
第三版，勉强能用。"

这不是炫技，这是**情绪的呼吸**。短句制造节奏，空拍让读者有时间消化信息。这种写法在"观点表达"上的作用是：让作者的态度通过节奏传递出来——着急、兴奋、沮丧，不用说出来，节奏本身就表达了。
