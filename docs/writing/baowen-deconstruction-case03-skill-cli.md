## 亮点拆解

### 1️⃣ 开头钩子：问题驱动，代入感强

> "有个群友问我：黄叔，CLI 和 Skill 到底谁调谁？是不是 CLI 是底层基础设施，Skill 是上层应用？这问题让我愣了好几秒。"

- 用**真实群友提问**开场，不是自说自话
- 承认"愣了好几秒"——**示弱即真诚**，拉近距离
- 问题的争议性天然勾起好奇心

### 2️⃣ 类比功力：菜谱和菜刀

> "它们不是上下级，不是谁包含谁。它们更像菜谱和菜刀。菜谱告诉你怎么切，菜刀帮你切。"

- 用**最日常的类比**讲透复杂概念
- 类比贯穿全文最后一段收尾呼应，形成闭环

### 3️⃣ 叙事节奏：故事拆概念

- 先讲**两个真实操作**（CLI调Skill、Skill封装CLI），用过程截图让读者"看到"
- 再抛出核心结论——**先让读者自己感受到矛盾，再给出答案**
- 不是"我告诉你答案"，是"你先看到问题，我们再一起想通"

### 4️⃣ 官方背书：不是野路子

- 挖出Anthropic官方文档的三个设计证据（scripts/、!command、allowed-tools）
- 论证"官方就是这么设计的"——让读者觉得"这不是hack，是正统玩法"
- 每个证据都用"第一个证据/第二个证据"带编号，**结构化输出增加说服力**

### 5️⃣ 判断口诀：可执行

> "手动三次，就该写成说明书。"

- 拒绝抽象，给出**具体可执行的判断标准**
- 比喻接续"连点三次同一家外卖，第三次该存进收藏夹"

### 6️⃣ 信息密度：多层读者都能获益

文章设计了三层读者：

| 读者层次 | 收获 |
|---------|------|
| 小白 | 分清CLI和Skill是什么 |
| 进阶 | 学会什么时候用谁、怎么互相嵌套 |
| 老手 | Skill链的组合玩法、下篇预告勾引 |

### 7️⃣ 结尾运营：引子 + 钩子 + 关注引导

> "下一篇，我讲一个更刺激的事…"
> "最近为啥老发 Skill 相关的内容呢？其实是积累的太久了…"
> "围观黄叔朋友圈：product2023"

- 留悬念勾住读者（下篇预告）
- 透露"后面还会发一份 Skill 蓝皮书"——价值承诺
- 结尾带个人微信号引流

## 可复用的写法

1. **问题开场法**：用「群友问了我一个问题」替代「今天聊聊xxx」
2. **三段递进**：实际操作 → 提炼概念 → 官方论证
3. **类比一以贯之**：选定一个类比就贯穿全文
4. **判断口诀**：把方法论压缩成一句话（"手动三次写成说明书"）
5. **下篇预告**：文中埋钩子增加关注率
6. **截图即证据**：操作类文章必须有截图，且是真实截图

## 风格特点（黄叔风格摘要）

- 语气：**技术男的真诚**，有干货但不装
- 人称：大量用"我"、"黄叔"（第三人称自称）
- 句式：短句为主，偶尔偏长但段落短
- 金句：输出类比和金句，但密度不高（约每2~3段一句）
- 结尾：必有下篇预告 + 个人微信号引流
- 信息来源：全部标注真实出处，建立信任
